# Плейбук: как сократить счёт за токены

Готовые приёмы, **сначала самые крупные выигрыши**. Каждый процент здесь — это либо реальное число, измеренное `token-diet bench` на встроенных датасетах, либо помеченная грубая оценка. Правило, которое бьёт любой приём: **измеряйте, а не гадайте** — запустите `token-diet diet your.json` на своих данных до выбора формата.

Все `%` ниже — относительно `json-pretty` (indent=2), счёт через `tiktoken cl100k_base`.

---

## 1. Табличная кодировка для массивов объектов

**Самый крупный выигрыш, когда вы отправляете списки записей.** Красивый и минифицированный JSON повторяет каждый ключ в каждой строке. Табличные форматы (заголовок `#N` в JTF или CSV) пишут ключи **один раз**, дальше — только строки.

Измерено на встроенных датасетах:

| Датасет | Форма | Лучший lossless | vs JSON |
|---|---|---|---:|
| `users` (50 плоских записей) | массив плоских объектов | `jtf` | **−62,9%** |
| `events` (40 аналитических событий) | массив объектов | `jtf` | **−27,9%** |

- **JTF** остаётся **lossless** (типы сохранены, реальный круговой обход) и умеет вложенные строки. Берите через [json-token-format](https://github.com/k1y0miiii/json-token-format).
- **CSV/TSV** ещё меньше (`users`: −67,5%), **но lossy** — в CSV нет типов, поэтому `36` становится `"36"`, а `false` — `"false"`. Годится, если данные и так строковые или модели сказано трактовать ячейки как строки; иначе предпочтите JTF.

Компромисс: нужен парсер/декодер на приёме. Не человекочитаемо.

## 2. Минификация JSON

Убрать отступы и пробелы между токенами: `json.dumps(x, separators=(",",":"))`. **Бесплатно, lossless, без смены парсера** — приёмник по-прежнему читает обычный JSON.

Измерено: **−34%…−41%** по встроенным датасетам (например, `config` −40,8%, `product_catalog` −40,9%). Это самое выгодное изменение, которое можно внедрить сегодня без риска.

```python
import json
payload = json.dumps(data, separators=(",", ":"), ensure_ascii=False)
```

Компромисс: чуть менее читаемо в логах. И всё.

## 3. Удалить null и пустые поля до отправки

Если поле — `null`, `[]`, `{}` или дефолт, который модели не нужен, опустите его. Lossy **намеренно** (вы решаете, что поле неважно), поэтому применяйте с умом.

```python
def prune(x):
    if isinstance(x, dict):
        return {k: prune(v) for k, v in x.items() if v not in (None, [], {}, "")}
    if isinstance(x, list):
        return [prune(v) for v in x]
    return x
```

Грубый эффект: **5…20%** в зависимости от разреженности данных. Сильнее всего — на ответах API, забитых необязательными `null`.

## 4. Сократить повторяющиеся ключи (с картой)

Замените длинные ключи (`emailAddress` → `e`) и приложите маленькую карту, чтобы остаться lossless. **Выигрыш только при сильном повторе ключей** — карта тоже стоит токенов, и `token-diet` считает её честно.

Измерено: `org_tree` **−41,4%** (глубокая вложенность, ключи повторяются тысячи раз), но на мелких и неглубоких данных карта обойдётся дороже экономии (`config`: **+9,5%** — реальная регрессия). Сначала измерьте.

Компромисс: не человекочитаемо; приёмник должен применить карту. Часто **минификация (приём 2) даёт почти то же без минусов** — берите короткие ключи только когда измерение подтвердило выгоду.

## 5. Веб-контент → чистый текст вместо сырого HTML

Сырой HTML — это в основном теги, атрибуты, скрипты и обвязка. Извлечение чистого текста/markdown до отправки в модель срезает **примерно в 8 раз** на типичных веб-страницах.

Используйте [glyph-mcp](https://github.com/k1y0miiii/glyph-mcp) — он забирает страницы и конвертирует их в чистый для LLM текст. Lossy (вы теряете разметку), но разметка редко нужна модели для рассуждения.

## 6. Кэширование промпта — кэшируйте стабильный префикс

Если вы отправляете один и тот же большой системный промпт/контекст на каждом вызове — кэшируйте его. Провайдеры считают кэшированные токены по доле от входной цены (и не пересчитывают их). **Огромная** экономия при повторных вызовах; выигрыш идёт по **цене**, а не по числу токенов, поэтому складывается со всем выше.

- Стабильное переиспользуемое содержимое — в **начало** промпта.
- Переменное, под конкретный запрос — в **конец**.

## 7. Гигиена контекста — отправляйте только нужное

Самый дешёвый токен — тот, который вы не отправили. Извлекайте и обрезайте до нужного среза вместо вываливания целых файлов, историй и схем. Эффект разный, но часто это самый большой рычаг — и он улучшает качество ответа, а не только цену.

## 8. Структурированный вывод — ограничьте ответ, а не только вход

Используйте режимы JSON-schema / structured output, чтобы модель отдавала ровно нужные поля без словесной обвязки. Срезает **выходные** токены (часто более дорогие) и убирает хрупкий разбор свободного текста.

---

## Измеряйте, а не гадайте

Один и тот же приём бывает выигрышем −63% или регрессией +9% в зависимости от формы данных (сравните `users` и `config` для коротких ключей выше). До того как стандартизировать формат:

```bash
token-diet diet your-real-payload.json
```

Для **точного счёта по Claude** (tiktoken здесь — приближение для семейства GPT) используйте [llmcost](https://github.com/k1y0miiii/llmcost) с `--api`.
